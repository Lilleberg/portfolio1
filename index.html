Community Science Museum

GitHub Repository Go to Community Science Museum

Changes made

  • Removed the gradient in the background.
  • Changed the cross-over between headings and navigation on Explore.html.
  • Added smooth scroll behavior.
  • Edited the width of the content on all pages.
  • Added alt text on the logo, as well as changed the logo a little bit.
  • Edited the navigation on desktop, and changed the hamburger menu from a checkbox to a button. The navigation will also close if tou click outside it on the main section.
  • Removed non-working search field.
  • Changed the contact form a little bit, and added form validation it.
  • Cleaned up the HTML a bit.

How the Changes Were Made

  • The heading/navigation cross-over were separated. Now the section navigation instead sticks to the top of the screen when scrolling.
  • The smooth scroll behavior was set on the html to work on the navigation mentioned above.
  • The width of the content was lessened, as it was a bit wide before.
  • The desktop navigation had an underline when active before, and a white background. Now it's more in style with the active page with colour and placement.
  • The previously used checkbox hack for the hamburger menu is now a button that uses JavaScript to work by an onclick function checking if it's displayed as block or none.
  • Form validation was added with JavaScript, checking that emails have the right format, and that the input fields have a minimum length.

Why I Made the Changes

  • Feedback on the project said the pages without gradient were easier to read. I agree.
  • Feedback said the cross-over between heading and navigation wasn't very intuitive, so that was changed.
  • The smooth scroll behavior was a tip I got from the feedback on the project, and it does indeed make it better.
  • The width of the content felt a bit too wide on desktops, which is why that was changed.
  • I changed the way the active links on the pages look because it was a bit boring. I feel like it now fits better with the target audience and the theme.
  • The form was changed to look a bit better. Form validation is also good to have on forms.
  • The HTML had some semantic problems: there were sections without headings; the fake links were paragraphs instead of links (feedback on project); the footer headings didn't use any heading tag, so I changed all that. On one page I had two H1's as well, now there's only one.
